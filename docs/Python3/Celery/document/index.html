Document

reference from: https://officeguide.cc/celery-distributed-task-queue-getting-started-1/
reference from: https://ithelp.ithome.com.tw/articles/10273022?sc=iThomeR

  • Celery 是一個以 Python 開發的分散式工作佇列排程系統，可以結合各類型的資料庫與 broker 處理即時或預定的排程工作，監控工作的排程狀況，等待計算完成之後再取回計算結果。
  • Celery 支援的 broker 有很多種，包含 RabbitMQ、Redis、Amazon SQS 等。
  • 在 Celery worker 執行之後，就會處於等待工作的狀態，它會負責接收工作，指派給 worker 進行處理。
  • 當Ｗorker做完會回傳AsyncResult結果，可以存在backend及透過PRC傳回程式。

Celery 架構圖#

install Celery#

pip3 install celery

建立worker service#

# 建立 Celery Application（以 RPC 送回工作狀態）#filename: tasks.pyfrom celery import Celeryapp = Celery('tasks', backend='rpc://', broker='redis://localhost/')
@app.taskdef add(x, y):    return x + y
# run servicecelery -A tasks worker --loglevel=INFO

建立tasks傳入worker取得結果#

#filename: new_task.pyfrom tasks import add
result = add.delay(4, 4)
# run codepython3 new_task.py
# 檢查工作是否已經完成result.ready()# 取得計算結果result.get()# 若不需要查看計算結果，則可呼叫 forget() 函數釋放儲存空間：# 釋放儲存空間result.forget()# 由於 backend 在儲存計算結果時也會需要一定的空間，所以對於每一個 AsyncResult 物件都一定要呼叫 get() 或是 forget() 函數，這樣這些計算結果所占用的儲存資源才會被釋放掉。

Celery 設定檔

# celeryconfig.py 設定檔內容
# Broker 與 Backend 設定broker_url = 'redis://localhost/'result_backend = 'rpc://'
# 其他設定task_serializer = 'json'result_serializer = 'json'accept_content = ['json']timezone = 'Asia/Taipei'enable_utc = False
# config app with confi using namespaceapp.config_from_object('django.conf:settings', namespace='CELERY')

app = Celery('name')
