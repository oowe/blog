解決無法生成超過 4G 的 iso 的問題#

Reference from https://www.itread01.com/content/1550058871.html

systemback 是一個很好的備份還原軟體。用它還可以生成自己的 linux 系統映象檔案。但是在使用時會發現當生成的 sblive 檔案大於 4G 的時候是沒有辦法生成 iso 檔案的。這是由於 iso 檔案自身的限制，iso9600 對於檔案有限制，單個檔案不能超過 2G，總的 iso 檔案不能超過 4G。

但是我們經常見到的系統安裝 iso 檔案很多都是超過 4G 的。這是怎麼回事呢。經過我的苦苦搜尋終於找到了一點線索。對於大於 4G 的可以採用 udf 檔案系統。下面就是具體的解決方法了。

#解壓 .sblive 檔案:mkdir sblive#停留在原資料夾位置即可tar -xf /home/systemback_live_2016-04-27.sblive -C sblive#重新命名 syslinux 至 isolinux:mv sblive/syslinux/syslinux.cfg sblive/syslinux/isolinux.cfgmv sblive/syslinux sblive/isolinux#安裝 cdtoolsaria2c -s 10 https://nchc.dl.sourceforge.net/project/cdrtools/alpha/cdrtools-3.02a07.tar.gz
tar -xzvf cdrtools-3.02a07.tar.gzcd cdrtools-3.02makesudo make install#生成 ISO 檔案:/opt/schily/bin/mkisofs -iso-level 3 -r -V sblive -cache-inodes -J -l -b isolinux/isolinux.bin -no-emul-boot -boot-load-size 4 -boot-info-table -c isolinux/boot.cat -o sblive.iso sblive

這樣就可以了，估計當檔案大於 4G 的時候程式會自動採用 udf 檔案系統，所以可以成功建立 iso 檔案。
