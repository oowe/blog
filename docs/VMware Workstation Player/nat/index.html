NAT

取得外部 IP,請選 Bridge

Virtual machine (虛擬機) 預設是利用 NAT (Network Access Translation 服務來上網

這個服務是 VM 自己有 DHCP 功能 發送一個 virtual IP 給虛擬機上網 可以進到虛擬機內查看 IP 應該會長成 192.168 網段的 IP

透過 NAT 的服務 可以對應到你本機上網的 IP 然後上網 所以你的虛擬機對外的 IP 和你本機的 IP 相同

NAT 服務本意是用來解決 IP 不足的問題，如果你有多餘的 IP 可以直接設定新的 IP，進行上網

如果你想利用虛擬機來架站可以參考這篇 Reference from https://blog.xuite.net/sphjlc062218/thinking/32175554-VMware%E6%9E%B6%E8%A8%AD%E4%BC%BA%E6%9C%8D%E5%99%A8%E5%85%81%E8%A8%B1%E5%A4%96%E9%83%A8%E9%80%A3%E5%85%A5%E8%A7%A3%E6%B1%BA%E6%96%B9%E6%A1%88(NAT+mode)

補充一下剛剛若要使用固定 IP 上網，要先到虛擬機設定內將網卡模式設定成 bridge(橋接)，以橋接模式連線，就像實體電腦直接連接數據機，而不透過本機。 剛實際測試這樣可以

架站架在 VM 裡面的流程，架站好後，在 VM 裡面試開網頁(確認 Web 服務有起來)，之後到 VM 的設定設定 port(確定要和 Web 的 port 相同)，再本機開看看是否可以連到 VM 所架的網頁，最後設定在本機設定 DDNS 服務，讓外部使用者可以依固定的網址連入該網站。

Gayway(閘道)定義：是本機對外的封包傳遞的下一個路徑。所以 VM 與本機相同的 Gayway，如果是 public IP 應該會是.254 結尾，如果是 private IP 應該是分享器的 IP .1 結尾，所以你要先查本機電腦的 Gayway。

DDNS 設定方式如下：
採用 no-ip 所提供的服務
(1)註冊帳號
(2)設定一個屬於自己的網域名稱
(3)安裝並設定 no-ip 軟體(用來對應浮動 IP)
(4)測試功能是否正常

Q:當同時灌 Virtrul Box 和 VMPlayer，為什麼設定 Bridge 會無法連線?
A:網路設定的 Config 會互相覆蓋修改，造成設定錯誤。
