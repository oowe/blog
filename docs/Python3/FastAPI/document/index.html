Question#

Q1: 為什麼使用FastAPI? Q2: 優點是什麼？ Q3: 缺點是什麼？ Q4: 和其他framework比較？

特性#

  • 快速
  • 自動生成 Swagger和redoc
  • Type Hint 支援，增加程式可讀性，檢查變數型態更方便
  • 解決 Python GIL 限制，可以跑 ASGI
    • WSGI: 同步溝通
    • ASGI: 非同步溝通

Middleware#

  • 稱為Middleware的原因，是因為介於frontend requests 到 backend api之間的軟件。

starlette#

  • 是一個web framework。

fastAPI

  • Uvicorn 可支援 HTTP/1.1 以及 WebSockets，但 HTTP/2.0 還不支援
  • 路由 Router 在上面的範例中有提到 router，這邊要提醒各位就是一定要在資料夾中加入 init.py 這個空檔案，在 Python 3 後倡導不需要這個東西，我們就很容易忽略這個傢伙！在 FastAPI 這是依靠他去找到對應的檔案，因此一定要先加上它，避免未來踩到雷。

example#

from fastapi import FastAPI
app = FastAPI()

@app.get("/")async def root():    return {"message": "Hello World"}

程式進入點#

uvicorn main:app --reload --port 8001
info

uvicorn <檔名>:<app名稱> --reload 在專案更新時，自動重新載入，類似Flask的debug模式 main：main.py文件（一個Python「模塊」）。 app：在 main.py 文件中通過 app = FastAPI() 創建的對象。

fastAPI特色#

  1. type hints (pydantic套件可以幫忙)

async function#

  1. 為非同步函數
  2. async-await

顺序很重要#

若是相同path，route的順序很重要。 example

@app.get('/books/my_book')async def read_favorite_book():    return {"book_title": "My Favorite Book"}

@app.get('/books/{book_id}')async def read_book(book_id: int):    return {"book_id": book_id}

default#

如果沒有增加敘述，則function就會變成敘述

depends#

  • Dependency Injection
  • path parameters
  • query parameters
  • body parameters
