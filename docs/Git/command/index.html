  5. git 都紀錄新的部分，舊的沒有更動就會指到上一個 commit

  7. git 內有四種物件，blob(放檔案)，tree(放目錄)，commit 及 tag

  15. git reset --mixed(default)->(放回工作區), --soft(將 commit 的內容放回索引區) --hard(直接放棄未 commit 的檔案)

git checkout test 和 origin/test 的差別

  • test 代表 local 的 branch，當選擇的同時他會指向遠方 reposity 的 branch HEAD
  • orgin/test 代表遠方 reposity 的 Head
  • 通常我們都會在本地的 branch 開發，在 git push 上遠方，同時 orgin/test/就會直接指向最新的 HEAD，不需要重新 git pull
  • 相反，若是從 orgin/test 的地方 git push 上遠方，test 並不會更新 等於 git push test 就會更新 origin/test
