建立 pyenv#

python3 -m venv python_envsource python_env/bin/activatepip3 install django

建立新的專案#

django-admin startproject mysitecd mysitepython3 manage.py runserver

概念#

1. 不同的功能就是不同的 app2. django-admin startproject <project_name> #建立新的專案3. python3 manage. py <command> #腳本可以創建應用，和資料庫通訊，啟動開發用網絡服務器4. python3 manage. py startapp catalog #創建 catalog app

startproject 後的資料夾(樹狀)#

├── asgi. py├── **init**. py (是一個空文件，指示 Python 將此目錄視為 Python 套件)├── settings. py （包含所有的網站設置。這是可以註冊所有創建的應用的地方，也是靜態文件，數據庫配置的地方，等等)├── urls. py （定義了網站 url 到 view 的映射。雖然這裡可以包含所有的 url，但是更常見的做法是把應用相關的 url 包含在相關應用中，你可以在接下來的教程裡看到)└── wsgi. py （幫助 Django 應用和網絡服務器間的通訊。你可以把這個當作模板

catalog（樹狀）#

├── admin. py (網站管理)├── apps. py (註冊應用)├── **init**. py (一個空文件，Django/Python 會將這個文件作為 Python 套件包並允許你在項目的其他部分使用它)├── migrations （當你修改你的數據模型時，這個文件會自動升級你的資料庫)│   └── **init**. py├── models. py (模型)├── tests. py (測試用)└── views. py (視圖函數

註冊服務和修改時區#

在 locallibrary 的 setting.py#addINSTALLED_APPS = ['django.contrib.admin','django.contrib.auth','django.contrib.contenttypes','django.contrib.sessions','django.contrib.messages','django.contrib.staticfiles',** 'catalog.apps.CatalogConfig', **]ChangeTIME_ZONE = 'Asia/Taipei'

#測試時期可以不用管，但正式上線要注意

  • SECRET_KEY. 這個密匙值，是 Django 網站安全策略的一部分。如果在開發環境中，沒有保護好這個密匙，把代碼投入生產環境時，最好用不同的密匙代替。（可能從環境變量或文件中讀取）。

  • DEBUG. 這個會在 debug 日誌裡輸出錯誤信息，而不是輸入 H​​TTP 的返回碼。在生產環境中，它應設置為 false，因為輸出的錯誤信息，會幫助想要攻擊網站的人。

manage.py 指令#

django-admin startproject 專案名稱  建立一個Django專案python manage.py startapp 應用名稱  建立一個APPpython manage.py runserver ip地址 port號   執行Django專案，如不指定IP與PORT預設為127.0.0.1與8000python manage.py makemigrations 建立資料庫的py檔案python manage.py migrate    將生成的py檔案應用到資料庫python manage.py flush  資料庫中資料全部清除，只留下表python manage.py createsuperuser    建立管理員使用者，密碼（郵箱可以留空）python manage.py changepassword 使用者名稱   修改管理員使用者密碼

Email Service#

reference from: https://www.learncodewithmike.com/2020/05/django-send-email.html

Q:RuntimeError: You called this URL via POST, but the URL doesn't end in a slash and you have APPEND_SLASH set. Django can't redirect to the slash URL while maintaining POST data. Change your form to point to localhost:8080/v3/email/resend/ (note the trailing slash), or set APPEND_SLASH=False in your Django settings.#

A: Django的規則是，URL必須要以/結尾。
