Unit Test - Gtest

Reference from https://ithelp.ithome.com.tw/articles/10202905
Reference from https://testing.googleblog.com/2008/07/tott-expect-vs-assert.html

  1. 全名為 google test，是 Google 所開發的 C++ test framework
  2. napmespace 為 tedting
  3. C++ test framework 也包含其他像是 CppTest、Check 等等

Install#

 #取得原始碼 apt install -y libgtest-dev #原始碼路徑 cd /usr/src/gtest #執行cmake sudo cmake CMakeLists.txt #編譯 sudo make #成功建置後會產生 libgtest.a 及 libgtest_main.a 兩個檔案

Example code#

#file name: example_gtest.cpp#include <gtest/gtest.h>
#define HELLOWORLD "Hello World"
TEST(GoogleTest, HelloWorld) {    ASSERT_EQ("Hello World", HELLOWORLD);}
int main(int argc, char **argv) {    testing::InitGoogleTest(&argc, argv);    return RUN_ALL_TESTS();}

編譯#

g++ example_gtest.cpp -o example_gtest -Igtest/include -Lgtest/lib -lgtest -lpthread

執行#

./example_gtest

常用語法#

#ASSERT vs EXPECT ASSERT:若有錯誤就會直接中止測試程式。 EXPECT:失敗後會繼續往下執行測試程式碼。 我覺得是使用情境不同的話，使用的語法不同。 舉例:若是三行不同的測試項目，那可以使用 EXPECT；若是連續的測試項目，建議使用 ASSERT。

//判斷 condition 為真ASSERT_TRUE(condition);EXPECT_TRUE(condition);
//判斷 condition 為假ASSERT_FALSE(condition);EXPECT_FALSE(condition);
//判斷結果相同為正確//格式為ASSERT_EQ(期待值，實際值)，為期望值和實際值判斷，若相同測試成功；若不相同測試失敗。ASSERT_EQ(expected,actual);EXPECT_EQ(expected,actual);
//判斷結果不同為正確ASSERT_EQ(expected,actual);EXPECT_EQ(expected,actual);
info
  1. 我猜判斷 condition 是為 bool 的判斷(待確認)
