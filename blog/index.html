Development#

  • 依開發流程所需得環境，分成DEV, Staging and Prod環境。
  • Dev: 撰寫期，系統開發所使用的環境。
  • Staging: 測試時期，QA進行測試的環境。
  • Prod: 正式上線，系統正式版上線的環境。

gitlab flow#

(網路方法1)

  • branch區分以develop, master及other三種。
  • 所有的branch都會進行CI。
  • 新功能在other branch開發完之後會發PR，當code review approved會merge到develop。
  • 接著，develop會進行CI，並CD部署到dev環境。
  • 當要上新版，會從develop branch發PR，當code review approved會merge到master。
  • 接著，master會進行CI。
  • master branch的CD會用dot tag的方式，手動觸發才會部署。

中文文件編號格式#

  1. 第一層編號為國字大寫(壹、貳、參、肆、伍)，不縮排
  2. 第二層編號為國字小寫(一、二、三、四、伍)，縮排兩字元，內文首行縮排兩字元
  3. 第三層編號為小括號及國字小寫組成，後面不得加頓號[(一))(二)(三)(四)(五)]，不縮排
  4. 第四層編號為阿拉伯數字及小數點組成(1. 2. 3. 4. 5.)，縮排兩字元，內文切齊不縮排

Reference from https://kknews.cc/zh-tw/n/y35z9vn.html

info

「一」「二」「三」和「甲」「乙」「丙」等，以及阿拉伯數字，用括號括起來，表示序次。帶括號的序次後面不能再用頓號或逗號。

英文文件編號格式#

Reference from https://www.editing.tw/IEEE%E8%8B%B1%E6%96%87%E8%AB%96%E6%96%87%E6%A8%99%E9%A1%8C-Title-%E6%91%98%E8%A6%81-Abstract-%E5%8F%83%E8%80%83%E6%96%87%E7%8D%BB-Reference%E6%A0%BC%E5%BC%8F

  1. 第一標題：以羅馬數字編號，置於段落的中間上方，所有字母大寫。
    =>羅馬數字(I, II, III, IV, V, VI, VII, VIII, IX, X)

  2. 第二標題：以大寫字母編號，遵循首字母大寫原則並斜體，靠左對齊。
    =>大寫字母(A,B,C,D,E)

  3. 第三標題：以阿拉伯數字編號，內縮行距為一個破折號的長度（Em Dash「—」，數字後加上冒號，倘若數字後面緊接標題，則在標題後加上冒號。標題文字的部分，則遵循首字母大寫原則。描述文字可直接在冒號後面，不需另起一行。
    =>阿拉伯數字(0,1,2,3,4,5)

  4. 第四標題：和第三標題格式相同，但是內縮行距為兩個破折號的長度（Em Dash「—」，標題文字遵循句首大寫原則。

Fitness

April 8, 2022 · 2 min read
oowe
Marker
  1. 降體脂肪重要的三個元素：喝水, 睡眠及飲食控制。
  2. 喝水：一天的需水量為體重*30~40 倍。
  3. 睡眠：至少七小時，可減少暴飲暴食的情況。
  4. 飲食控制：一天一餐，延長空腹的時間。
  5. 讓一日一餐和餓了才吃變成自己的生活習慣。

反思#

  1. 吃牛肉湯麵，不要點麵(精緻澱粉量太大)，湯還在評估要不要喝。
  2. 吃披薩，單點一個就好，且隔天體重會重；大餐能自己選擇的時候盡量不要選披薩。
  3. 假日會吃兩餐，單餐就不能吃的和平日一日一餐豐盛。
  4. 體重降 10%會遇到停滯期，這個時候體重就不會下降這麼快，因為身體正在進行再平衡，請保持原有的飲控和運動習慣。

Oxygen Management System

March 24, 2022 · 3 min read
oowe
Marker

個人練習專案

Status Define#

  1. 全新品:全部條件都不符合為全新品
  2. 保固中:出貨紀錄(機器序號)有出現及出貨紀錄(保固起始日期)未滿一年。
  3. 待修復:出保紀錄(原本機器序號)有出現且機器狀態(已修復)不等於是。
  4. 二手品:出保紀錄(原本機器序號)有出現且機器狀態(已修復)等於是。
  5. 過保固:出貨紀錄(機器序號)有出現及出貨紀錄(保固起始日期)已滿一年。

目的#

提供下單管理，入庫管理，出貨管理，保固維修等項目進行管理系統

專案功能#

  1. 取代現行 word 紙本紀錄方式
  2. 精準掌握下單，現貨，出貨及保固維修狀態
  3. 需保護買家客資
  4. 使用人為管理者 1 人

大功能及子項目#

  • 登入頁面:
    • 登入頁面，設定一組帳號密碼
  • 下單管理:
    • 廠商資料總表，建立/修改/刪除廠商資料
    • 下單資料總表，建立/修改/刪除下單資料
  • 現貨管理:
    • 入庫資料總表，建立/修改/刪除入庫資料
    • 現貨資料總表，建立/修改/刪除現貨資料
  • 出貨管理:
    • 出貨資料總表，建立/修改/刪除出貨資料
  • 保固維修:
    • 保固維修資料總表，建立/修改/刪除保固維修資料

希望的網頁界面樣式#

  1. 左邊欄選擇項目
  2. 右邊顯示內容
  3. 大項目點開才會看到子項目

資料庫設計#

  • Database name: oms

  • 登入頁面

    • table name: user

  • 下單管理

  • 現貨管理

  • 出貨管理

  • 保固維修

商業 SOP#

  • 廠商: 單一廠商，品質較穩定。
  • 寄送方式: 以海運為主，價格便宜。
  • 保固: 我方提供賣家保固。
