單元測試精神#

單元測試側重於為盡可能小的單元編寫測試。在大多數情況下，它們是在類中定義的函數。一個類中的 MethodA 可能正在調用另一個類中的 MethodB。但是，MethodA 的單元測試只關注 MethodA 的邏輯，而不關注 MethodB。單元測試不應該依賴於它們運行的環境，它們應該是快速的。要編寫隔離的單元測試，通常會模擬方法/服務的所有依賴項

測試域#

beforeAll ：所在區域內會第一個執行。 beforeEach ：每一次的測試前會先執行。 afterAll ：所在區域內最後一個執行。 afterEach ：每一次的測試後會馬上執行。

parmeter#

  • .toBe: 需要完全符合
//要測試的函式const sum = (a, b) => {  return a + b;};
test("Check the result of 5 + 2", () => {  //在 expect 中呼叫函式測試結果  expect(sum(5, 2)).toBe(7);});
  • .toMatch: 搭配正規表達式檢查
test("Use toMatch test", () => {  //搭配正規表達式  expect("Happy new year.").toMatch(/new/);});
  • .toEqual: 針對『物件』等於期望值
test('Check the object type', () => {    let peopleA = {        name: 'GQSM'    }    peopleA.age = 25
    //測試字串    expect(peopleA.name).toBe('GQSM')    //測試物件    expect(peopleA).toEqual({ name: 'GQSM', age: 25 })})
  • .not: 預防函式回傳某個結果
test("Use not", () => {  let peopleA = {    name: "GQSM",  };  //確認 name 不等於空  expect(peopleA.name).not.toBe("");  peopleA.name = "";  //如果 name 是空的  expect(peopleA.name).not.toBe("");});
  • .toContain: 是針對陣列的元素，只要有出現就算對
test('For array test in jest',()=>{    let arrA = ['A','B','C']
    //檢查陣列內是否含有某值    expect(arrA).toContain('B')
    //搭配迴圈檢查每個位置都不等於空    for(let i in arrA){        expect(arrA[i]).not.toBe('')    }})
  • 判斷 undefined 、 null 、 true
test("Special value", () => {  //期望值為 true  expect(true).toBeTruthy();
  //期望值為 false  expect(false).toBeFalsy();
  //期望值為 null  expect(null).toBeNull();
  //期望值為 undefined  expect(undefined).toBeUndefined();
  //期望值為 undefined 之外的值  expect(null).toBeDefined();});

設計自動化測試流程在 DCS account Module#

基本條件

  1. 每一個 function 都需要 token 才能做
  2. 要做 function 包含 (1) GET: Get account list (2) POST: Create new account (3) DELETE: Delete an account

程式如下:

  1. 自動化測試先拿到 token
  2. 先做 Get account list: 目的要可以拿到目前資料庫的所有帳號 list。
  3. 再做 Create new account: 目的要確定這個功能可以新增帳號，是不是新增完之後在單查 get account by name 的這個 RESTful api 就可以驗證這個功能。
  4. 最後做 Delete an account: 目的要確定這個功能可以新增帳號，是不是新增完之後在單查 get account by name 的這個 RESTful api 就可以驗證這個功能。

解決 Jest 在 test 時找不到 module 問題 (20210701 順利解決)#

error message: Cannot find module 'src/account/account.dto' from 'auth.service.ts'

Reference from https://github.com/jest-community/vscode-jest/issues/382

因為在 package.json 內，jest 會自定義路徑，要去更改其內容 請依照下方內容修改

- rootDir 要由"src"改成"./"- 增加"modulePaths": ["<rootDir>"],
    "jest": {        "moduleFileExtensions": [            "js",            "json",            "ts"        ],
        "rootDir": "./",        "modulePaths": [            "<rootDir>"          ],        "testRegex": ".spec.ts$",        "transform": {            "^.+\\.(t|j)s$": "ts-jest"        },        "coverageDirectory": "../coverage",        "testEnvironment": "node"    }
