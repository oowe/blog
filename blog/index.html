個人練習專案

Status Define#

  1. 全新品:全部條件都不符合為全新品
  2. 保固中:出貨紀錄(機器序號)有出現及出貨紀錄(保固起始日期)未滿一年。
  3. 待修復:出保紀錄(原本機器序號)有出現且機器狀態(已修復)不等於是。
  4. 二手品:出保紀錄(原本機器序號)有出現且機器狀態(已修復)等於是。
  5. 過保固:出貨紀錄(機器序號)有出現及出貨紀錄(保固起始日期)已滿一年。

目的#

提供下單管理，入庫管理，出貨管理，保固維修等項目進行管理系統

專案功能#

  1. 取代現行 word 紙本紀錄方式
  2. 精準掌握下單，現貨，出貨及保固維修狀態
  3. 需保護買家客資
  4. 使用人為管理者 1 人

大功能及子項目#

  • 登入頁面:
    • 登入頁面，設定一組帳號密碼
  • 下單管理:
    • 廠商資料總表，建立/修改/刪除廠商資料
    • 下單資料總表，建立/修改/刪除下單資料
  • 現貨管理:
    • 入庫資料總表，建立/修改/刪除入庫資料
    • 現貨資料總表，建立/修改/刪除現貨資料
  • 出貨管理:
    • 出貨資料總表，建立/修改/刪除出貨資料
  • 保固維修:
    • 保固維修資料總表，建立/修改/刪除保固維修資料

希望的網頁界面樣式#

  1. 左邊欄選擇項目
  2. 右邊顯示內容
  3. 大項目點開才會看到子項目

資料庫設計#

  • Database name: oms

  • 登入頁面

    • table name: user

  • 下單管理

  • 現貨管理

  • 出貨管理

  • 保固維修

商業 SOP#

  • 廠商: 單一廠商，品質較穩定。
  • 寄送方式: 以海運為主，價格便宜。
  • 保固: 我方提供賣家保固。

Survey OpenADR

我國現行電力環境面臨以下問題對於備轉容量造成危機：#

  1. 區域供電瓶頸
  2. 核電將逐年除役
  3. 再生能源易受氣候因素影響
  4. 新穩定能源開發不易

我國推動需量反應歷史:#

  1. 民國 68 年起開始推動 DR(Demand Respone)
  2. 民國 109 年起推動 OADR，但仍未普及。

OpenADR 歷史#

起源 Electric Power Research Institute(美國電力研究所 release).

介紹#

  • 什麼是 OpenADR

    • 全名: Open Automated Demand Response
    • 中文全名:自動需求回應
    • 最新版本: 2.0b
    • 協定目標在於標準化，自動化電力網路之需量反應(DR)電能調度，以使公用事業商與聚合商能夠高效地管理不斷增長的能源需求與分散式能源。
    • 分成兩個部份:
      • Virtual-End-Node(VEN): 地端的角色
      • Virtual Top Node(VTN): 台電的角色
    • OpenADR 定義了機器對機器的接口並包括信息模型，傳輸和安全機制，以及在兩個端點之間交換數據的方式。OpenADR 2.0 定義了電力服務提供商和客戶之間的信息通信內容和方式，但沒有定義任一端點如何使用信息。
    • OpenADR 基本功能:事件管理(Event)，程式管理(Program)，VEN 管理，自我測試及除錯。
    • VTN 和 VEN 溝通是利用 XMPP，是一種以 XML 為基礎的開放式即時通訊協定

  • Service

OADR 功能#

  1. EiRegistor 部分
  • Registor VEN to VTN
  • VTN 建立允許的 VEN 清單
  • VTN 有一個 list 可知目前的連線清單
  1. EiEvent 部分
  • VTN 能建立事件並指定發送給某 VEN
  • VTN 會收到 VEN 是否加入的回應
  • 建立兩個事件，VEN 一個同意一個拒絕
  1. EiOpt 部分
  • VEN 回應某事件是否要加入
  1. EiReport 部分
  • VEN 可以定期回報資訊給 VTN

VTN 功能#

  1. VTN 配置(vtn_name, port, url 自定義)
  2. 註冊新的 VEN 設備
  3. VEN 清單查詢
  4. 選顯示所選擇的 VEN 內容 (ven_name, ven_id, ven_fingerpoint)
  5. 發送 event
  6. 與 VENs 通訊紀錄

角色定義：#

  1. 電力服務提供者(台電)
  2. 用戶群代表
  3. 終端使用者

預期的功能(透過 Restful API)#

  1. 可以註冊新的 VEN
  2. 可以修改特定 VEN
  3. 可以刪除特定 VEN
  4. 可以查詢特定事件
  5. 可以修改特定事件
  6. 可以刪除特定事件
  7. 可以要求 VEN 報告
  8. 可以查詢特定報告
  9. 可以修改特定報告
  10. 可以刪除特定報告
  11. 可以傳送加入事件
  12. 可以傳送拒絕事件

部屬方式#

  1. 用單機執行
  2. Container 化
