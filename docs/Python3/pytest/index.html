Pytest

  • 單元測試
  • pytest預設會執行目錄下 test.py及test.py的測試檔案。
  • pytest {filename} 執行也可以

單一測試#

poetry run pytest ./{project}/tests/api/test_api.py -m "not jwt"

@pytest.mark.asyncio()#

Reference from: https://stackoverflow.com/questions/57461868/for-what-pytest-mark-asyncio-is-used

@pytest.mark.asyncioasync def test_example(event_loop):    do_stuff()        await asyncio.sleep(0.1, loop=event_loop)
equal to
def test_example():    loop = asyncio.new_event_loop()    try:        do_stuff()        asyncio.set_event_loop(loop)        loop.run_until_complete(asyncio.sleep(0.1, loop=loop))    finally:        loop.close()

@pytest.mark.parametrize建立多個測試參數值#

Reference from https://ithelp.ithome.com.tw/articles/10227361 名稱要對應到參數名稱 

def sum(num1,num2): return num1 + num2
import pytest
@pytest.mark.parametrize( 'num1,num2,eq_num', [  (1,2,3),  (4,5,9),  (0,0,0) ])
def test_multi_sum(num1,num2,eq_num): assert sum(num1,num2) == eq_num

pytest --co#

只收集測試Function列表，不做測試。

@pytest.fixture()#

@pytest.fixture() 如果不寫參數，參數默認 scope='function'
