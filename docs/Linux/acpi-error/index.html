ACPI Error

Error message#

“: [DSSP] Namespace lookup failure, AE_NOT_FOUND (20160831/psargs-359)[ 2.673960] ACPI Error: Method parse/execution failed [_SB.PCI0.SAT0.SPT4._GTF] (Node ffff906e950b7370), AE_NOT_FOUND (20160831/psparse-543”.

解決方式#

Reference from https://askubuntu.com/questions/160036/how-do-i-disable-acpi-when-booting

原因：因為主機板認不得顯示卡，所以無法啟動報 error解決方法：在 grub 時，進設定將 acpi=off 輸入即可
Please add acpi=off to the kernel command line -- it's just slightly different than doing it with the LiveCD:
1. Press the Shift when booting up, to get the Grub menu. Use the arrow keys to navigate/highlight the entry you want (usually the first one).2. Press e to edit that entry, which will show you the details:3. Find the linux entry as shown above, use the arrow keys to get to it, and then press the End key to get to that line's end (which may be on the next line!). Enter acpi=off as shown, and press Ctrl+X to boot.4. You can make this workaround permanent by:   - sudo nano /etc/default/grub   - Change line GRUB_CMDLINE_LINUX_DEFAULT="quiet splash" to GRUB_CMDLINE_LINUX_DEFAULT="quiet splash acpi=off"   - Ctrl-X, press Y and then Enter to save and exit.   - sudo update-grub   -
