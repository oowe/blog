ts在重構很方便你可以在function上面按F2改名字，有使用到的地方都會跟著更改
註解的方式/***   isPositive 為檢測輸入值是否為正數的函數*   ＠param input: number 為輸入的數字*   @output boolean 為輸出的結果，如果為正數，則輸出為true*/

型別宣告的空白沒有限制前後一定要空白但通常是user: string (:後面空一格)
let和var的差別ES6 新增了let命令，用来声明变量。它的用法类似于var，但是所声明的变量，只在let命令所在的代码块内有效
基本資料型別JavaScript 的型別分為兩種：原始資料型別（Primitive data types）和物件型別（Object types）。原始資料型別包括：boolean (布林值)、number (數值)、string (字串)、null、undefined 以及 ES6 中的新型別 Symbol。
字串如果要含變數通常都會這樣加let sentence: string = `Hello, my name is ${myName}. I'll be ${myAge + 1} years old next month.`;其中 ` 用來定義 ES6 中的範本字串，${expr} 用來在範本字串中嵌入表示式。
JavaScript 沒有空值（Void）的概念，在 TypeScript 中，可以用 void 表示沒有任何返回值的函式function alertName(): void {    alert('My name is Tom');}
any 型別，則允許被賦值為任意型別變數如果在宣告的時候，未指定其型別，那麼它會被識別為任意值型別：let something;something = 'seven';something = 7;something.setName('Tom');#以下會報錯let something = 'seven';something = 7;
聯合型別（Union Types）表示取值可以為多種型別中的一種。let a: string | number;a = 10;a = 'Aaron';console.log(a);
在 TypeScript 中，我們使用介面（Interfaces）來定義物件的型別在物件導向程式語言中，介面（Interfaces）是一個很重要的概念，它是對行為的一種抽象，而具體如何行動則需要由類別（classes）去實現（implement）。TypeScript 中的介面是一個非常靈活的概念，除了可用於對類別的一部分行為進行抽象以外，也常用於對「物件的形狀（Shape）」進行描述。
interface Person {    name: string;    age: number;}
let tom: Person = {    name: 'Tom',    age: 25};上面的例子中，我們定義了一個介面 Person，接著定義了一個變數 tom，它的型別是 Person。這樣，我們就約束了 tom 的形狀必須和介面 Person 一致。介面一般首字母大寫。
可選屬性有時我們希望不要完全匹配一個形狀，那麼可以用可選屬性 (?)：interface Person {    name: string;    age?: number;}
let tom: Person = {    name: 'Tom'};

有時候我們希望一個介面允許有任意的屬性，可以使用如下方式：interface Person {    name: string;    age?: number;    [test: string]: any;}
let aaron: Person ={    name: 'aaron',    age: 35,    tall: 190};console.log(aaron);
唯讀屬性有時候我們希望物件中的一些欄位只能在建立的時候被賦值，那麼可以用 readonly 定義唯讀屬性：interface Person {    readonly id: number;    name: string;    age?: number;    [propName: string]: any;}
let tom: Person = {    id: 89757,    name: 'Tom',    gender: 'male'};

Difference between 'let' and 'const'#

  • 參數是固定的請用 const
  • 參數是可變的請用 let
  • var 是屬於固定或可變動格式，較不嚴謹，不建議使用。
let outputNumber = 10;console.log("a = " + outputNumber);outputNumber = 30;console.log("a = " + outputNumber);
const outputWord = "Practice";console.log("a = " + outputWord);outputWord = "fail"; //Errorconsole.log("a = " + outputWord);

IF ELSE#

let a: number = 200;let b: number = 200;
if (a > b) {  console.log("a is more then b");} else if (a < b) {  console.log("a is less then b");} else {  console.log("a is equal then b");}

FOR LOOP#

function add(max: number): number {  let sum = 0;  for (let i = 0; i <= max; i++) {    sum = sum + i;  }  return sum;}let output: number = add(10);console.log(output);

different with 'let' and 'var'#

作用域不一樣，var 的作用域在函數 (function) 裡，let 的作用域則是在區塊 (block) 裡。

interface 和 class 差異#

  • interface 比較像檢查器，進來的資料要符合這個規範
  • class 是你宣告了一個物件，你要定義它的屬性。
