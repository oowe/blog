Oxygen Management System

March 24, 2022 · 3 min read
oowe
Marker

個人練習專案

Status Define#

  1. 全新品:全部條件都不符合為全新品
  2. 保固中:出貨紀錄(機器序號)有出現及出貨紀錄(保固起始日期)未滿一年。
  3. 待修復:出保紀錄(原本機器序號)有出現且機器狀態(已修復)不等於是。
  4. 二手品:出保紀錄(原本機器序號)有出現且機器狀態(已修復)等於是。
  5. 過保固:出貨紀錄(機器序號)有出現及出貨紀錄(保固起始日期)已滿一年。

目的#

提供下單管理，入庫管理，出貨管理，保固維修等項目進行管理系統

專案功能#

  1. 取代現行 word 紙本紀錄方式
  2. 精準掌握下單，現貨，出貨及保固維修狀態
  3. 需保護買家客資
  4. 使用人為管理者 1 人

大功能及子項目#

  • 登入頁面:
    • 登入頁面，設定一組帳號密碼
  • 下單管理:
    • 廠商資料總表，建立/修改/刪除廠商資料
    • 下單資料總表，建立/修改/刪除下單資料
  • 現貨管理:
    • 入庫資料總表，建立/修改/刪除入庫資料
    • 現貨資料總表，建立/修改/刪除現貨資料
  • 出貨管理:
    • 出貨資料總表，建立/修改/刪除出貨資料
  • 保固維修:
    • 保固維修資料總表，建立/修改/刪除保固維修資料

希望的網頁界面樣式#

  1. 左邊欄選擇項目
  2. 右邊顯示內容
  3. 大項目點開才會看到子項目

資料庫設計#

  • Database name: oms

  • 登入頁面

    • table name: user

  • 下單管理

  • 現貨管理

  • 出貨管理

  • 保固維修

商業 SOP#

  • 廠商: 單一廠商，品質較穩定。
  • 寄送方式: 以海運為主，價格便宜。
  • 保固: 我方提供賣家保固。
